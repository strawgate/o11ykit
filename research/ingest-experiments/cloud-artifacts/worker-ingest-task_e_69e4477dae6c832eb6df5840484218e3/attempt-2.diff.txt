+function createWorkerClient() {
+  const worker = new Worker(join(pkgRoot, 'dist', 'worker.js'), { type: 'module' });
+  const workerLike = {

+      worker.postMessage(message, transfer);

+async function measureEventLoopLagWhile(fn) {
+  let running = true;
+  let ticks = 0;
+  let maxLagMs = 0;
+  let totalLagMs = 0;
+  const intervalMs = 1;
+  let expected = performance.now() + intervalMs;
+
+  const timer = setInterval(() => {
+    const now = performance.now();
+    const lag = Math.max(0, now - expected);
+    if (lag > maxLagMs) maxLagMs = lag;
+    totalLagMs += lag;
+    ticks += 1;
+    expected = now + intervalMs;
+  }, intervalMs);
+
+  const start = performance.now();
+  const result = await fn();
+  const elapsedMs = performance.now() - start;
+
+  await new Promise((resolve) => setTimeout(resolve, 10));
+  running = false;
+  if (running === false) clearInterval(timer);
+
+  return {
+    result,
+    elapsedMs,
+    maxLagMs,
+    avgLagMs: ticks > 0 ? totalLagMs / ticks : 0,
+  };

+# 04 — Worker ingest experiment results
+
+Date: 2026-04-19
+
+## Prototype summary
+
+Implemented worker-offloaded OTLP ingest (parse + flush) using the existing worker runtime/client architecture:
+
+- Added a new worker `ingest` request/response carrying OTLP JSON payload and returning full `IngestResult`.
+- Preserved existing typed-array batch append path by renaming it to explicit `append` operation.
+- `WorkerClient.ingest(payload: string)` now UTF-8 encodes payload and transfers the underlying `ArrayBuffer` to worker when using transferable strategy.
+- Worker runtime decodes UTF-8 bytes and calls `ingestOtlpJson(payload, store)` entirely inside worker.
+
+## Benchmark setup
+
+Script: `research/ingest-experiments/04-worker-bench.mjs`
+
+Measured 30 iterations with 10,000 gauge points per payload:
+
+1. **Sync path**: `ingestOtlpJson(payload, storage)` on main thread.
+2. **Worker path**: `WorkerClient.ingest(payload)` round trip.
+3. Event-loop lag sampler (`setInterval` at 1ms) used as main-thread blocking proxy.
+

+## Interpretation
+
+- **Main-thread blocking improves significantly** with worker ingest (~54ms → ~9ms avg in this Node benchmark harness).
+- **Round-trip latency increases** (worker + message overhead), and throughput drops by ~18% in this prototype.
+- This is consistent with expectations: we trade peak single-thread throughput for better UI responsiveness.
+
+## Exploration notes
+
+### SharedArrayBuffer for zero-copy reads
+
+Potentially useful, but not immediate:
+
+- SAB could avoid some copies between worker and main thread for query results or ingest payload staging.
+- In browsers, SAB requires cross-origin isolation headers (COOP/COEP), which adds deployment constraints.
+- Storage internals are not currently designed as a shared-memory ring; adopting SAB would need protocol + memory-layout redesign.
+
+### Backpressure behavior
+
+Current client sends unbounded requests; if producer rate exceeds worker processing rate:
+
+- pending promises and queued messages will grow,
+- memory usage can climb,
+- tail latency will worsen.
+
+Suggested follow-up:
+
+- add `maxInFlight`/queue limit to `WorkerClient`,
+- expose `ingestQueueDepth` metrics,
+- optionally support dropping/coalescing policy for non-critical streams.
+
+### Can `append()` be replaced by worker ingest?
+
+Not fully.
+
+- `append()` remains useful when caller already has parsed typed arrays and labels.
+- Worker `ingest()` is better when caller has raw OTLP JSON and wants main-thread isolation.
+- Both operations should coexist: different input contracts and optimization points.
+
+## Recommendation
+
+**OPTIONAL** (opt-in API now, not default yet).
+
+Rationale:
+
+- Strong main-thread responsiveness improvement validates the architecture.
+- Throughput/latency overhead needs browser-focused tuning before making default.
+- Next step should be browser benchmark + explicit backpressure controls; if results hold under real UI load, promote to default.
