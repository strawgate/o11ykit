+# Experiment 01: bypass `flattenAttributes` allocation in ingest hot path
+
+## Goal
+Remove the per-point intermediate `Record<string, unknown>` allocation in the OTLP ingest hot path by adding a zero-allocation attribute iteration API in `@otlpkit/otlpjson` and using it from `packages/o11ytsdb/src/ingest.ts`.
+
+## Implementation summary
+- Added `forEachAttribute(attributes, onAttribute)` to `@otlpkit/otlpjson`.
+  - Iterates `OtlpKeyValue[]` directly.
+  - Converts each `OtlpAnyValue` via existing `attributeValueToJs`.
+  - Avoids creating a top-level flattened object when caller only needs iteration.
+- Kept `flattenAttributes` unchanged for existing callers and API compatibility.
+- Updated ingest hot-path consumers to use direct iteration:
+  - `computePointAttrsHash(baseHash, pointAttrs)` now iterates OTLP attrs directly.
+  - `buildSnapshotLabels(baseEntries, metricName, pointAttrs)` now iterates OTLP attrs directly.
+- Removed the old `cachedFlattenAttributes` object cache and replaced it with a lighter hash result cache keyed by `(pointAttrs reference, baseHash)`.
+  - This preserves fast paths for repeated attribute references while avoiding object materialization on misses.
+
