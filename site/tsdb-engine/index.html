XOR-Delta Compression

Based on Facebook's Gorilla paper (VLDB 2015). Timestamps use delta-of-delta with zigzag encoding. Values use XOR with leading/trailing zero tracking.

1

Timestamp: Delta-of-Delta

Regular intervals compress to 1 bit per sample. Irregular intervals use 9–68 bits with zigzag encoding.

DoD == 0            → '0'              (1 bit)
DoD in [-63, 64]    → '10' + 7-bit zz  (9 bits)
DoD in [-255, 256]  → '110' + 9-bit zz (12 bits)
DoD in [-2047,2048] → '1110' + 12-bit  (16 bits)
Otherwise           → '1111' + raw 64  (68 bits)
2

Value: XOR Encoding

Identical values: 1 bit. Similar values reuse the leading/trailing zero window from the previous sample.

XOR == 0         → '0'              (1 bit)
Reuse window     → '10' + meaningful (2 + N bits)
New window       → '11' + 6L + 6M   (14 + N bits)
3

Result: Massive Compression

Constant-interval gauges achieve 57× compression vs raw Float64. Even noisy data compresses 4–8×.

Quick Start

Use the TSDB engine in three steps: create a store, ingest data, and query.

import { FlatStore, ChunkedStore, ColumnStore, ScanEngine, encodeChunk, decodeChunk } from 'o11ytsdb';

// 1. Create a storage backend
const store = new ChunkedStore({ name: 'xor-delta', encode: encodeChunk, decode: decodeChunk });

// 2. Ingest time-series data
const labels = new Map([['__name__', 'http_requests'], ['method', 'GET'], ['region', 'us-east']]);
const seriesId = store.getOrCreateSeries(labels);

const timestamps = BigInt64Array.from({ length: 10000 }, (_, i) => BigInt(Date.now() - (10000 - i) * 1000) * 1_000_000n);
const values = Float64Array.from({ length: 10000 }, (_, i) => 100 + Math.sin(i / 50) * 30 + Math.random() * 5);
store.appendBatch(seriesId, timestamps, values);

// 3. Query with aggregation
const engine = new ScanEngine();
const result = engine.query(store, {
  metric: 'http_requests',
  start: timestamps[0],
  end: timestamps[timestamps.length - 1],
  agg: 'avg',
  groupBy: ['region'],
  step: 10_000_000_000n, // 10-second buckets
});

console.log(`Found ${result.series.length} groups, scanned ${result.scannedSamples} samples`);

OTLP Ingest Pipeline

Ingest OpenTelemetry JSON payloads directly. Supports gauge, sum, histogram, summary, and exponential histogram metric types.

import { ingestOtlpJson, ChunkedStore, ScanEngine } from 'o11ytsdb';

const store = new ChunkedStore(codec);

// Ingest raw OTLP metrics JSON
const result = ingestOtlpJson(otlpPayload, store);
console.log(`${result.pointsAccepted} points, ${result.seriesCreated} new series`);
console.log(`Types: ${JSON.stringify(result.metricTypeCounts)}`);
// → { gauge: 42, sum: 18, histogram: 6, summary: 2 }

// Labels are auto-extracted:
//   resource.service.name, scope_attr.*, attr.*, __name__
// Histogram generates derived series:
//   metric_bucket, metric_count, metric_sum

Web Worker Isolation

Move all TSDB operations off the main thread. The worker protocol supports structured clone, transferable buffers, and SharedArrayBuffer strategies.

import { WorkerClient } from 'o11ytsdb';

const client = new WorkerClient(new Worker('./tsdb-worker.js'));
await client.init({ chunkSize: 1024 });

// Ingest from main thread — data transfers automatically
await client.ingest(labels, timestamps, values);

// Query returns results back to main thread
const result = await client.query({
  metric: 'http_requests',
  start: 0n,
  end: BigInt(Date.now()) * 1_000_000n,
  agg: 'avg',
  step: 10_000_000_000n,
});

const stats = await client.stats();
console.log(`${stats.seriesCount} series, ${stats.sampleCount} samples, ${stats.memoryBytes} bytes`);
