Product docs: Octo11y

Turn GitHub activity into telemetry you can graph.

Octo11y is for repository maintainers and platform teams that want to convert workflow logs, benchmark outputs, and repo metadata into structured run history and published views.

Where Octo11y Fits

Who Uses It

OSS maintainers, CI owners, and developer productivity teams who want objective trend data across pull requests and releases.

What You Get

A clean data branch with run artifacts, indexes, and per-metric series files that can be consumed by dashboards, badges, and custom reports.

Core Capabilities

Parse Results

Read benchmark outputs from files or workflow logs and convert to OTLP metrics.

Aggregate History

Produce run indexes, refs/PR views, and per-metric time-series artifacts.

Repository Metrics

Collect stars, forks, workflow health, and language distribution as telemetry.

Quickstart Workflow

name: benchmark-pipeline
on: [workflow_dispatch]

permissions:
  contents: write
  actions: read

jobs:
  run-benchmarks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: go test ./... -bench=. | tee bench.txt
      - uses: strawgate/o11ykit/octo11y/actions/parse-results@main-dist
        with:
          mode: file
          results: bench.txt
          format: go
          github-token: ${{ github.token }}
      - uses: strawgate/o11ykit/octo11y/actions/aggregate@main-dist
        with:
          github-token: ${{ github.token }}

Typical Outcomes

  • Trend charts for benchmark and repository metrics over time
  • Consistent metric history per branch, PR, and commit
  • Data artifacts that power public or internal pages
