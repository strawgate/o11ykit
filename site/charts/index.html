user docs

What the adapter gives you.

Query the TSDB engine once, then pass the result directly to the adapter for the chart library you already use. The adapter returns native package input: Tremor props, Recharts rows and keys, Chart.js configs, ECharts options, Plotly figures, uPlot aligned arrays, Vega-Lite specs, and the equivalent shape for each supported library. Use the Show code button on any card to compare the query, adapter call, returned input, and native render code.

ergonomics audit

What you still control.

The adapters handle the repetitive conversion work: aligned timestamps, latest values, histogram buckets, labels, sparse values, and package-specific field names. You keep control over presentation, colors, tooltips, axes, and the update strategy each chart library recommends. A chart shape only appears in this gallery when that library has a published adapter and a real native renderer path for it.

Labels and nulls

Supply a `seriesLabel` callback and let the adapter preserve sparse samples as the null or missing-value shape that library expects.

seriesLabel: (series) => ...

Styling remains native

The adapter gives you the data-bearing object. You can still merge in normal chart package options for themes, legends, grids, cursors, and interaction.

{ ...option, grid, tooltip }

Updates are package updates

For live data, append into the same store, query the sliding window, convert again, then call the chart library's own update API.

result -> adapter -> chart.update()

why it feels good

Each adapter speaks the library's dialect.

The public exports are named for the thing the library wants, not for an o11ykit intermediate model. A Tremor user gets props, a Recharts user gets rows and data keys, uPlot gets aligned arrays, ECharts gets options, and Plotly gets a figure.
