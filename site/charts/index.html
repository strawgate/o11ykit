first wave

Adapters feel like the chart library.

Every demo starts from the same engine result and then switches into the library's preferred surface: Tremor props, Recharts row models, Chart.js configs, ECharts datasets, or uPlot aligned arrays.

Tremor

Prop-first components

Line, area, bar, donut, and bar list demos expose readable categories, nullable gaps, value formatters, and metadata for drilldown. 

const wide = toEngineWideTableModel(result, {
  seriesLabel: (s) => s.labels.get("host") ?? "host",
});
const props = toTremorLineChartProps(wide);
Recharts

Rows plus data keys

Recharts demos keep the model simple: rows for chart data, series descriptors for rendering lines, bars, areas, composed charts, and latest-value views. 

const wide = toEngineWideTableModel(result);
const model = toRechartsEngineTimeSeriesModel(wide, {
  unit: "ms",
});
Next wave

Native update models

The gallery plan tracks the chart types, lazy update path, and demo state for each library so Chart.js, ECharts, uPlot, and the wider ecosystem can land cleanly. 

const line = toChartJsLineConfig(frame);
chart.data = line.data;
