TSDB engine, as a primary building block

If you need more than transient in-memory views, the TSDB engine is the storage and compression layer for browser-native telemetry products.

Two interchangeable runtimes

Use the pure TypeScript codec for simpler deployments and lower-performance paths, or switch to Rust→WASM when the hot path needs much higher throughput.

More than one storage shape

The engine already spans chunked and columnar backends, with XOR-delta, ALP, and Delta-ALP codecs selected per chunk.

Built to stay explainable

The engine site includes live demos plus codec walkthroughs, so the storage layer stays teachable instead of becoming an opaque binary dependency.

Explore the engine Codec walkthroughs Source

Keep OtlpKit and o11ytsdb adjacent, not merged

OtlpKit is the OTLP shaping layer. o11ytsdb is the compression and storage layer. They fit together naturally, but the current split keeps the storage engine reusable and lets the TypeScript implementation remain the low-friction option when WASM would be overkill.
